GitもRubyもやったことない人間が興味本位でOctopressを始めようとしたので、 驚く位ブログを立ち上げるだけで時間をとられてしまった。
後で、導入方法を記述する予定。
HTML は特にバリバリ書いているのでわかるものの、Octopress で記事を作成するには Markdown なる記述方法が必要とのこと。 なので調べてみたことを記述。 珍しくWikipediaがナンセンスだったので、備忘録がてら記述〜。

Markdownとは？自分解釈

  • 軽量マークアップ言語 (結局マークアップ言語のHTMLに直す)
  • 簡単
  • 意味付けのテキストが少ない
  • class id をつけられない＝結局HTML使うらしい。

Markdown Markup 違い

  • Markup … <タグ>sample</タグ>
  • Markdown … #記号 #h1見出し ##h2見出し 以降記号と表現する。

お固い解釈など興味のある方は 本家へ http://daringfireball.net/projects/markdown/

改行の後に半角スペース2回以上入れる。

強調 em strong

強調を示す場合はアスタリスクで囲むか
アンダースコアで囲む emは＊か＿で一つ囲む
strongは ＊か＿で二つ囲む

引用 q

引用な文章です。

入れ子にすることもできる。

リスト番号なし ul > li

リストを示すには*(アスタリスク),–(ハイフン),+(プラス)のどれかを入力し、スペースまたはタブで挿入する。

  • 上下に空白を一つ入れよう
  • 記号の前後に空白をつけると見れなくなるよ
  • 記号の後にスペースをお忘れなく

リスト番号あり ol > li

感想

細かな設定は結局 HTML なるものの、新しい面白い技術。
vim + Zencoding を積んでカスタマイズしても、Markdownの方が、楽に文章を書くことができる。
割とvimとは相性が良くない方かもだけど、vimrcにスにペッドを書いてもよさそう。
とりあえず、スペースの可視化は必須かな〜
